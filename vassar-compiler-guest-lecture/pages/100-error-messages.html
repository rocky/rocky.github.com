Parsing with Error Positions

Programming tip: when giving an error message, try to provide what was expected as well as what was given.

Now that we have captured positions from the scanner and parser, we should use those in error messages.

I won't go over the code for this. You can find it on github. It is not that difficult or interesting.

But I will show a test run to show what we get when we now have position information available [PLAY]

I think the error messages could look a little nicer. I leave this as an exercise for you to do this if you want to do so.

But I do want to mention in writing a good error messages to try to provide two pieces of information: what the computer expected, and what the computer got instead.
