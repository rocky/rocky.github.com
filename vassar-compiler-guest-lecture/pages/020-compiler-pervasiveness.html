The Pervasiveness of Compilers


My laptop user process tree building Python 3.13

When I was getting my computer-science degree in college, there were a number courses on topics that did not have much bearing on my current day-to-day use of computers. However, I use a compiler virtually every day I program.

This screen shot shows a snapsot of the user processes on my laptop in building Python 3.13. New releases come out 3 or 4 times a year.

The stuff in bright green on the right, [Right-CTRL Left click to Zoom], These are currently running process taking up CPU time. You might not be able to read the process names, but most of those green processes say "cc1", which is early phase of running the "gcc" compiler toolchain. You'll also see one assembler process "Ayee-Ess" which is the GNU x86 assembler, running a later phase of gcc tool chain.

Popular e-commerce sites that are using continuous integration and deployment.

And even if you are not a computer scientist or programmer, when you browse an e-comerce site that is under constant development, like Amazon.com, compilers are continually getting run in the process of testing and deploying code to web-sites.

And even if the website did not run a compiler to build and test the site, the browser on your laptop or mobile phone may have a compiler running that is turning Javascript into WASM (Web assembly). It does this to make your browser work faster.
