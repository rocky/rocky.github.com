What are Containers?

Some possibilities:

  • a file in a file system,
  • a stream from a socket or piped into a compiler,
  • a piece of an "archive"-format file: Java classfile jar, Python wheel (zip), or machine-code library, etc.,
  • a string or AST data-structure created inside a program at runtime, e.g. HTML templating system for Web framework, an AI auto-learning system

Program source code comes from somewhere. It is useful for ta programmer or an IDE to know where it came from. So what are some possibilities? As I said before, a file in the file system is the most common possibility. Any other suggestions? [Solicit ideas...]

a piece of an archive-format file. In Java this is called "class file" or jar file and is encoded like a zip file. Python wheels are zip files too. Machine code libraries are collections too, they are in a format called "archive" and are built using the command Aee R in Linux. Note: the word, "jar" used by Java the word is a modification of (Aee v R) in machine code. So "jar" stands for "Java archive" and you may have heard about a "tar" file which stands for "tape archive" even though nowadays physical tapes are no longer used.

Another possibility is a string or AST data-structure created inside a program at runtime, for example an HTML templating system for Web framework, or AI self-modying code in an auto-learning system.

In the 331 language, programs come from a file stored in your file system. So we won't dwell further on the containers. But note that while locations inside a container change bit, the container itself doesn't change that often. This is useful to keep in mind when designing a data structure for positions.

In the 331 language, there is only one container. The standard runtime data format which we will talk about later, provides for file path. It also understands that file paths are contained inside a directory.
